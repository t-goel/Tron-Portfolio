## Directory Purposes

**src/:**
- Purpose: All application code
- Contains: React components, state management, configuration data, styling
- Key organization: By functional domain (3D, UI, store, data)

**src/components/:**
- Purpose: Reusable React components
- Contains: Scene orchestration, 3D geometry/lighting, 2D UI overlays
- Subdirs: 3D (Three.js), UI (DOM overlays), phases (future phase components)

**src/components/3D/:**
- Purpose: Interactive Three.js/React Three Fiber components
- Contains: IdentityDisc (procedurally textured disc with particles and grid)
- Key files:
  - `IdentityDisc.jsx`: Main interactive element with procedural texture generation, particle system, grid floor, hover animations

**src/components/UI/:**
- Purpose: 2D DOM overlays rendered on top of WebGL canvas
- Contains: Title text with glitch effects, future: nav buttons, modals, status displays
- Key files:
  - `TitleOverlay.jsx`: Name and subtitle with glitch animation on disc hover

**src/components/phases/:**
- Purpose: Reserved for phase-specific full-screen components
- Current: Empty (placeholder for Boot Sequence, Main Hub, Shatter & Dock, Grid World, Sector Dives)
- Will contain: One component per phase, conditionally rendered by Scene.jsx

**src/data/:**
- Purpose: Centralized portfolio content configuration (not rendered directly by these files)
- Contains: Project metadata, skill categories, contact information
- Pattern: ES6 exports of arrays/objects with standardized schemas
- Key files:
  - `projects.js`: Array of 3 placeholder projects with id, name, tagline, techStack, accentColor, githubUrl, position
  - `skills.js`: Array of 5 skill category objects (languages, frameworks, tools, AI tools, certifications)
  - `contact.js`: Object with github, linkedin, email URLs

**src/store/:**
- Purpose: Zustand global state management
- Contains: Phase machine state, audio toggles, UI visibility flags, sector navigation
- Key files:
  - `appState.js`: Zustand store with subscribers for phase, hudVisible, activeSector, audioEnabled

**src/assets/:**
- Purpose: Static media files (fonts, textures, models — future)
- Contains: TrustMe font file (Tron-JOAa.ttf)
- Key files:
  - `assets/fonts/Tron-JOAa.ttf`: Custom Tron-inspired font for headers

**public/:**
- Purpose: Served unchanged to browser (root-level assets)
- Contains: index.html, favicon, any static assets needed at /

**dist/:**
- Purpose: Vite production build output
- Generated by: `npm run build`
- Contains: Bundled JS, CSS, optimized assets
- Not committed to git (in .gitignore)

## Key File Locations

**Entry Points:**
- `src/main.jsx`: First file executed; mounts App to DOM
- `src/App.jsx`: Root React component; creates Canvas + overlay structure
- `public/index.html`: HTML page that loads Vite bundle (standard CRA pattern)

**Configuration:**
- `vite.config.js`: Vite bundler settings (React + Tailwind plugins)
- `eslint.config.js`: Linting rules and globals
- `package.json`: Dependencies, build scripts, project metadata

**Core Logic:**
- `src/components/Scene.jsx`: Phase-conditional rendering, lighting setup
- `src/components/3D/IdentityDisc.jsx`: Procedural geometry, particle animation, user interaction
- `src/store/appState.js`: Global state store (phase, audio, UI)

**Testing:**
- Not applicable (no test files yet; `.test.js` or `.spec.js` would go co-located with source)

**Content Data:**
- `src/data/projects.js`: Project portfolio entries
- `src/data/skills.js`: Skill categories and competencies
- `src/data/contact.js`: Contact links

## Naming Conventions

**Files:**
- PascalCase for React components: `IdentityDisc.jsx`, `TitleOverlay.jsx`, `Scene.jsx`
- camelCase for utility/data files: `appState.js`, `projects.js`, `skills.js`
- kebab-case for directories where structure reflects content: `3D/`, `UI/`, already established but minimal

**Directories:**
- Domain-based organization: `components/`, `data/`, `store/`, `assets/`
- Functional groupings within: `components/3D/`, `components/UI/`, `components/phases/`
- Flat structure for config: root level for vite.config.js, eslint.config.js

**Functions & Variables:**
- camelCase for all functions, variables, and object keys
- CONST_CASE for constants: `DISC_RADIUS`, `TEX_SIZE`, `GLITCH_CHARS`, `GLITCH_CHARS`
- Single underscore prefix for refs that persist across renders: `discGroupRef`, `gridRef`, `glitchInterval`

**React Hooks & State:**
- `use` prefix for custom/standard hooks: `useAppState`, `useState`, `useFrame`, `useRef`
- Descriptive names for state vars: `hovered`, `displayName`, `activeSector`

## Where to Add New Code

**New Phase Component:**
- Primary code: `src/components/phases/{PhaseName}.jsx`
- Pattern:
  - Export default function component
  - Accept phase-conditional props from Scene.jsx
  - Use useAppState hook to read/write state
  - Render 3D content (R3F) or UI (DOM)
  - Add conditional rendering in Scene.jsx: `{phase === N && <PhaseComponent />}`

**New 3D Element (Interactive or Static):**
- Implementation: `src/components/3D/{ElementName}.jsx`
- Pattern:
  - Use R3F (`useFrame`, `useThree`, `useRef`)
  - Use Three.js directly for geometries and materials
  - Procedural content with `useMemo` for performance
  - Interactive: implement `onPointerEnter`, `onPointerLeave`, `onClick` handlers
  - Export as default function component
  - Import and render in Scene.jsx or parent component

**New UI Overlay Component:**
- Implementation: `src/components/UI/{ComponentName}.jsx`
- Pattern:
  - Positioned absolutely within overlay div
  - Use inline styles or Tailwind classes
  - No 3D rendering (DOM only)
  - Subscribe to appState if needed: `const state = useAppState(s => s.property)`
  - Example: `TitleOverlay.jsx` is the template

**New Data Configuration:**
- Implementation: `src/data/{category}.js`
- Pattern:
  - ES6 named export: `export const categoryName = [...]`
  - Array of objects with consistent schema
  - Each entry tagged with id, position (if spatial), colors, metadata
  - Example: `skillCategories`, `projects`, `contact`
  - Import where needed: `import { categoryName } from '../data/category'`

**Utilities & Helpers:**
- Shared functions: `src/utils/{utility}.js` (directory not yet created, add if needed)
- Pattern: Functional utilities, no state
- Import: `import { helper } from '../utils/utility'`

**Custom Hooks:**
- Reusable stateful logic: `src/hooks/use{HookName}.js` (directory not yet created)
- Pattern: Export function starting with `use`, may use other hooks internally
- Example: Could refactor particle animation logic from IdentityDisc into `useParticleSystem`

## Special Directories

**dist/:**
- Purpose: Build output directory
- Generated: Yes (by `npm run build`)
- Committed: No (listed in .gitignore)
- Contents: Minified JavaScript, CSS bundles, optimized assets
- Regenerate: `npm run build` before deployment

**node_modules/:**
- Purpose: Installed npm packages
- Generated: Yes (by `npm install`)
- Committed: No (listed in .gitignore)
- Contents: All dependencies from package.json

**public/:**
- Purpose: Static files served at URL root
- Generated: No (manually created)
- Committed: Yes
- Usage: Place index.html, favicon, robots.txt here

**assets/ (src/assets/):**
- Purpose: Media files bundled with app
- Generated: No (manually added)
- Committed: Yes
- Contents: Fonts, textures, 3D models (when added)
- Pattern: Imported as URLs in components: `url('./assets/fonts/Tron-JOAa.ttf')`
