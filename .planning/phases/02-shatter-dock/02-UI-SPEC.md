## Interaction Contracts

### Transition Sequence (Phase 2 → 3)

Full ordered sequence from "ENTER THE GRID" click to HUD settled state:

1. `EnterButton` onClick → `setPhase(3)` → `TitleOverlay` + `EnterButton` unmount (phase === 2 guard lifts)
2. Simultaneously: DOM disc div mounts at `calc(50vh - 60px) / calc(50vw - 60px)` (px-based center)
3. Simultaneously: `GridFloor` mounts in Scene.jsx (`phase >= 3` guard) and begins 0 → 0.6 opacity fade over 2s
4. After `delay: 0.05s`: GSAP tween starts — DOM disc lerps to `top: 24px, left: 24px, scale: 0.35` over 1.2s with `power2.inOut` easing
5. GSAP onComplete fires: `setHudVisible(true)` → DOM disc unmounts, HUD home div mounts with `.hud-disc` at fixed position
6. "TANMAY GOEL" text opacity 0 → 1 over 200ms with 200ms delay after HUD mounts
7. HUD controls div mounts (bottom-right) — social icons stagger fade in over 400ms

### HUD Home Button Click

- Fires: `setPhase(2)` (signals return to title screen)
- Visual: no animation on the button itself; phase change will unmount grid and show TitleOverlay (full NAV-02 camera lerp deferred to Phase 3)

### MuteToggle Click

- Fires: `toggleAudio()` Zustand action
- Visual: icon swaps immediately between SpeakerWave / SpeakerXMark (React re-render)
- No fade or delay on icon swap

### SocialIcon Click

- Fires: opens URL in new tab
- Visual: scale 1.1 + intensified glow on hover (CSS, no JS needed); no transition on click

## Animation Timing Reference

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|

## Accessibility Contract

| Concern | Requirement |
|---------|-------------|
| Touch targets | MuteToggle and all SocialIcons: minimum 44x44px (WCAG 2.5.5 AAA) |
| Keyboard navigation | MuteToggle is a `<button>` — receives focus naturally. SocialIcons are `<a>` — keyboard navigable. |
| Screen reader | All icon-only buttons carry aria-label. No icon has decorative text that needs aria-hidden. |
| Focus ring | No custom focus style required for Phase 2 (Tron aesthetic: focus ring would be visually conflicting). Defer to browser default. |
| Reduced motion | Grid floor fade and disc dock animation proceed regardless (no `prefers-reduced-motion` handling in Phase 2 scope). |

## CSS Additions to index.css

/* Phase 2: HUD disc idle rotation */
@keyframes discSpin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }

.hud-disc {
  width: 40px;
  height: 40px;
  border-radius: 50%;
