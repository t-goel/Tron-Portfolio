**3. Update the existing useFrame to handle decrypt progress and ring animation:**

After the existing billboard rotation code, add:

```js
// Decrypt progress update
const d = decryptRef.current
if (d.direction !== 0) {
  const speed = d.direction > 0 ? delta / 1.2 : delta / 0.6
  d.progress = Math.max(0, Math.min(1, d.progress + d.direction * speed))
  if (d.progress <= 0 || d.progress >= 1) d.direction = 0
}

// Ring pulse animation
const rs = ringStateRef.current
if (rs.active && ringRef.current) {
  rs.progress = Math.min(rs.progress + delta / 0.8, 1)
  const s = 0.5 + rs.progress * 4.5
  ringRef.current.scale.setScalar(s)
  ringRef.current.material.opacity = 0.8 * (1 - rs.progress)
  if (rs.progress >= 1) {
    rs.active = false
    ringRef.current.visible = false
  }
}
```

Modify the existing texture throttle section to call `drawFrame(ctx, seed, decryptRef.current.progress, label)` instead of `drawIdleFrame(ctx, seed)`. When `d.direction !== 0` (actively animating), bypass the 125ms throttle — draw every frame for smooth decrypt animation. When idle (direction === 0 and progress === 0), keep the 125ms throttle.

**4. Add pointer event handlers on the mesh element:**

<mesh
  onPointerEnter={(e) => {

Since the ring is outside the `<group>` (which has `position` prop and billboard rotation), and GatewayPane uses `forwardRef` forwarding to `groupRef`, the component return must wrap both `<group>` and `<mesh>` (ring) in a React fragment `<>...</>`. **Important notes:**

- Use `scale.setScalar(s)` not `scale.set(s, s, 1)` per research pitfall #4 to avoid ellipse distortion.
- The ring mesh uses `visible={false}` by default and is set to `visible={true}` on pulse fire.
