## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, character spacing units |
| sm | 8px | Gap between boot log lines (internal line padding) |
| md | 16px | Padding inside hint pill, boot log container horizontal padding |
| lg | 24px | Vertical offset from screen edge for affordance hint (bottom: 80px rounds to nearest token — see exception) |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions: The affordance hint is positioned `bottom: 80px` — this is 20px above the HUD disc (which sits at the bottom edge). 80px is 5×16px (5 md tokens), acceptable as a layout-specific override. Do not change to 64px or 96px — 80px clears the HUD disc with 8px breathing room.

Source: RESEARCH.md — `GridAffordanceHint` architecture pattern specifies `bottom: '80px'`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (`--void-black`) | BootSequence background (full-screen `fixed inset-0`), body/root background |
| Secondary (30%) | `rgba(0, 255, 255, 0.08)` | No panel/card UI in this phase — boot screen is full-bleed black |
| Accent (10%) | `#00FFFF` (`--neon-cyan`) | Boot log text color, affordance hint text color, glow effect on both |
| Destructive | `#FF0000` (`--crimson-red`) | Not used in Phase 6 — no destructive actions |

Accent reserved for: boot log text lines, affordance hint text, text-shadow glow on both. Accent is NOT used for backgrounds, borders, or any other element in this phase.

Glow specification:
- Boot log lines: `text-shadow: 0 0 6px #00FFFF` — subtle terminal phosphor glow
- Affordance hint: `text-shadow: 0 0 8px #00FFFF` — slightly stronger to read against 3D scene background (RESEARCH.md architecture pattern specifies this value exactly)

No white or light backgrounds anywhere in Phase 6. The existing `bg-[#fafafa]` in `BootSequence.jsx` is the primary artifact being replaced. Confirmed: `html, body, #root` already has `background: var(--void-black)` in `index.css` (line 29).

Source: CLAUDE.md design system, `src/index.css` CSS variables, RESEARCH.md pitfall warnings.

## Component Inventory

Two components are in scope for Phase 6. No third-party components.

### 1. BootSequence.jsx (FULL REWRITE — BOOT-05)

**File:** `src/components/UI/BootSequence.jsx`
**Export contract:** `export default function BootSequence({ onComplete })` — unchanged
**Mount point:** Already wired in `App.jsx` as `{showBoot && <BootSequence onComplete={handleBootComplete} />}` — no App.jsx changes for the component interface

**Visual spec:**
- Outer wrapper: `position: fixed`, `inset: 0`, `z-index: 50`, `background: #000` — full-screen void-black layer
- Boot log container: centered vertically and horizontally, `padding: 0 16px`, max-width 480px, left-aligned text
- Each line: `font-family: 'Roboto Mono', monospace`, `color: #00FFFF`, `font-size: 0.75rem`, `line-height: 1.8`, `text-shadow: 0 0 6px #00FFFF`
- Lines animate in via CSS `@keyframes bootLineIn` staggered with `animation-delay: ${i * 0.25}s`
- Component fades out: CSS `opacity` transitions from 1 → 0 over 500ms before `onComplete()` fires

**Boot log lines (exact copy):**
```
GRID OS v7.2.1 — INITIALIZING...
Loading kernel modules...
Mounting /dev/grid0 ... [OK]
Calibrating light cycle array...
Authenticating user identity...
Neural bridge established.
Welcome to the Grid.
```

**Timing contract (CRITICAL — audio must be preserved):**
- T+0ms: mount, call `initAudio()`, attach `playerror` handler
- T+800ms: call `playWithFade(2000)`
- T+2500ms: begin fade-out (`setFadingOut(true)`)
- T+3000ms: call `onComplete()` — App.jsx unmounts component

**CSS keyframe to add to index.css:**
```css
@keyframes bootLineIn {
  from { opacity: 0; transform: translateY(4px); }
  to   { opacity: 1; transform: translateY(0); }
}
```

**Fade-out class behavior:** When `fadingOut` state is true, outer wrapper transitions `opacity: 0` over `0.5s ease`. Use inline style, not a class, to keep it co-located with the component.

### 2. GridAffordanceHint.jsx (NEW — PANE-02)

**File:** `src/components/UI/GridAffordanceHint.jsx`
**Export contract:** `export default function GridAffordanceHint()`
**Mount point:** `App.jsx` — render when `{hudVisible && !activeSector && <GridAffordanceHint />}`

**Visual spec:**
- Container: `position: fixed`, `bottom: 80px`, `left: 50%`, `transform: translateX(-50%)`, `z-index: 25`, `pointer-events: none`
- Text: `font-family: 'Roboto Mono', monospace`, `color: #00FFFF`, `font-size: 0.75rem`, `letter-spacing: 0.2em`, `text-transform: uppercase`, `text-shadow: 0 0 8px #00FFFF`
- No background fill, no border, no pill/badge — text only with glow

**Interaction/animation spec:**
- Appears instantly (no fade-in) — opacity: 1 on mount
- Auto-fades: at T+3000ms set `fadingOut: true` → CSS `transition: opacity 0.6s ease` → opacity 0
- Auto-removes: at T+3600ms set `visible: false` → `return null`
- sessionStorage gate key: `'grid_hint_shown'` — set to `'1'` on first render, check before showing

**Session gate behavior:**
- First grid visit in session: hint appears, key written, auto-fades
- Any subsequent grid visit in same session: `sessionStorage.getItem('grid_hint_shown')` is truthy → component returns null immediately, no animation
- After tab close: sessionStorage cleared, hint shows again on next first visit

## Interaction States

### BootSequence States

| State | Visual |
|-------|--------|
| `idle` (T+0 to T+0ms) | Black screen, no text yet |
| `playing` (T+0 to T+2500ms) | Lines animate in one by one, staggered 0.25s apart |
| `fading-out` (T+2500ms to T+3000ms) | Whole component opacity transitions to 0 |
| `done` | Component returns null — App.jsx has taken over |

### GridAffordanceHint States

| State | Visual |
|-------|--------|
| `hidden` (already seen this session) | Returns null immediately — never renders |
| `visible` (T+0 to T+3000ms) | Full opacity, text with cyan glow |
| `fading-out` (T+3000ms to T+3600ms) | CSS opacity transition 0.6s ease → 0 |
| `removed` (T+3600ms+) | Returns null |
