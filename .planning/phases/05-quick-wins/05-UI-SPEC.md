## Interaction Contract

This section is specific to Phase 5 — all three changes are behavioral, not structural new UI.

### PANE-01 — Idle Pane Labels

| Property | Value |
|----------|-------|
| Element | Canvas 2D texture inside `GatewayPane.jsx` `drawFrame` |
| State: at rest | `rgba(0,255,255,0.75)` at 18px TR2N, centered at (120, 80) on 240×160px canvas |
| State: hover/decrypt active | Existing decrypt animation unchanged — label rendered by the existing decrypt path |
| State: fully decrypted | Existing behavior unchanged |
| Transition | None — canvas redraws at ~8fps; no CSS transition required |

Before (current): `rgba(0,255,255,0.35)` at 14px — too faint to read at rest
After (required): `rgba(0,255,255,0.75)` at 18px — legible at rest, stays dimmer than hover state to preserve "reveal" reward

### TITLE-01 — Fast Title Clear

| Property | Value |
|----------|-------|
| Element | Outer `<div>` in `TitleOverlay.jsx`, `style.transition` |
| Current value | `'opacity 1s ease-in'` |
| Required value | `'opacity 0.15s ease-out'` |
| Behavior | On CTA click: opacity goes 1 → 0 in 150ms. `ease-out` starts immediately visible (snappy), decelerates cleanly. Title must be visually gone before 250ms. |
| Mount/unmount | Do not change — component stays mounted while `phase >= 2`. Only transition duration changes. |

### NAV-04 — ESC Exits Sector

| Property | Value |
|----------|-------|
| Element | `window` keydown listener in `App.jsx` |
| Trigger | `e.key === 'Escape'` |
| Guard | Only fires when `useAppState.getState().activeSector !== null` |
| Action | `useAppState.getState().setActiveSector(null)` |
| Visual feedback | None beyond existing sector exit transition (sector fade/unmount already handled by App render logic) |
| Anti-pattern | Do NOT bind inside sector components — listener must survive component unmount |
