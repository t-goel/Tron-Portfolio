**1. Create ProjectsSector overlay (src/components/UI/ProjectsSector.jsx):**

CRITICAL — Pointer-events layering for Monolith hover through overlay:
- **Outer container (backdrop):** `position: 'fixed'`, `inset: 0`, `zIndex: 30`, `pointerEvents: 'none'`. This is the full-viewport backdrop layer. Setting `pointerEvents: 'none'` lets pointer events pass through to the R3F canvas underneath, enabling Monolith `onPointerEnter`/`onPointerLeave` hover detection.
  - Background: `rgba(0,0,0,0.85)`, `backdropFilter: 'blur(8px)'`, `WebkitBackdropFilter: 'blur(8px)'`
  - Fade-in on mount: use `useState(false)` for `visible`, `useEffect(() => { requestAnimationFrame(() => setVisible(true)); }, [])`, CSS transition `opacity 0.4s ease-in`
- **Inner content wrapper:** `pointerEvents: 'auto'`, `overflowY: 'auto'`, `overflowX: 'hidden'`, `height: '100%'`. This re-enables pointer events ONLY on the scrollable content area, so project cards, links, and scroll are all interactive. The wrapper sits inside the outer container.
  - `maxWidth: 680px`, `margin: '0 auto'`, `padding: '32px 16px'` (xl top/bottom, md sides)

This two-layer approach means:
  - Pointer events that miss the inner content wrapper fall through to the canvas (Monolith hover works)
  - Pointer events on cards/links are captured by the inner wrapper (UI stays interactive)
