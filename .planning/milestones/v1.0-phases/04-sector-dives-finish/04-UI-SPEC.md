## Spacing Scale

Declared values (multiples of 4 only). Phase 4 introduces full-viewport HTML overlays — spacing tokens govern these overlay panels directly in addition to the existing HUD.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tech tag inline padding, terminal glyph gaps, node label inset |
| sm | 8px | Tag gap between pill tags, terminal line gap between command and output |
| md | 16px | Card internal padding (project card), terminal horizontal padding, sector overlay edge inset |
| lg | 24px | Vertical gap between project cards, terminal command-group gap |
| xl | 32px | Sector overlay top/bottom padding, mobile card vertical spacing |
| 2xl | 48px | Reserved — sector title-to-content gap on wide viewports |
| 3xl | 64px | Reserved — not used in Phase 4 |

Exceptions:
- HUD disc is fixed at 40×40px (pre-existing from Phase 2, src/index.css `.hud-disc`) — not on the 8-point scale, must not be altered.
- Three.js world units (camera positions, monolith geometry) are independent of CSS px spacing tokens — not mapped here.
- HTML Canvas 2D Skills graph: Tier 1 node radius = 30px, Tier 2 node radius = 18px, radial formation radius = 200px. These are canvas render-context units, not CSS spacing.
- Terminal typewriter: `border-left: 2px solid` for project card accent — 2px is the established Tron fine-line convention (matches HUD disc border and pane EdgeGeometry).
- Mobile breakpoint: `<768px` — matches established CSS pattern across all phases.

## Interaction Contract

### Sector Navigation Transition (PROJ-01, ABOUT-01, SKILLS-01)

Source: 04-CONTEXT.md — fully specified.

- **Trigger:** `onPointerDown` on a `GatewayPane` mesh
- **Guard:** Zustand `transitioning: true` flag set on click; prevents double-clicks during animation
- **Step 1 (0–0.5s):** All 3 gateway panes fade out — `opacity 1 → 0` over 0.5s (GSAP)
- **Step 2 (0–1.0s):** Camera lerps to position 2 units past the clicked pane center — `ease-in-out`, duration 1.0s (GSAP)
- **Step 3 (0.8–1.2s):** Sector component mounts and fades in — `opacity 0 → 1` over 0.4s
- **Step 4:** `setActiveSector('projects' | 'about' | 'skills')` dispatched to Zustand; `transitioning: false` restored
- **Return path:** Clicking top-left HUD disc → `setActiveSector(null)` + reverse GSAP lerp to `[0, 8, 14]` + panes fade back in

### Projects Sector — 2D Scrollable Overlay (PROJ-02/03/04/05/06)

Source: 04-CONTEXT.md decisions.

- **Container:** fixed `100vw × 100vh`, `overflow-y: auto`, `backdrop-filter: blur(8px)`, background `rgba(0,0,0,0.85)`
- **Scroll axis:** vertical only — `overflow-x: hidden`
- **Card layout:** vertically stacked column, centered, max-width `680px`, centered horizontally
- **Card structure:**
  - Left border: `border-left: 2px solid {accentColor}` — Crimson Red for `active: true`, Neon Orange for all others
  - Project name: TR2N font, Display size (`clamp(2rem, 5vw, 4rem)`), color `--off-white`, `letter-spacing: 0.3em`
  - Tagline: Roboto Mono, Label size (`clamp(0.7rem, 1.5vw, 1rem)`), color `rgba(240,240,240,0.8)`
  - Tech stack: row of inline pill tags — `14px Roboto Mono`, `border: 1px solid rgba(0,255,255,0.4)`, `background: rgba(0,255,255,0.08)`, `border-radius: 4px`, `padding: 4px 8px`, color `--neon-cyan`
  - "IN PROGRESS" badge (active projects only): top-right corner of card, `14px Roboto Mono`, `border: 1px solid --crimson-red`, `background: rgba(255,0,0,0.15)`, `color: #FF0000`
  - "VIEW ON GITHUB ↗" link: CTA size TR2N, `color: rgba(0,255,255,0.7)` at rest → full `#00FFFF` + `text-shadow: 0 0 10px #00FFFF` on hover, `target="_blank"`, `rel="noopener noreferrer"`
- **Decorative 3D background:** monolith silhouettes remain visible behind the overlay as dark geometry — no additional blur on the 3D scene (overlay backdrop-filter handles depth cue)
- **Data source:** `src/data/projects.js` — executor must prompt user to populate real data before testing

### About Me Sector — Terminal Interface (ABOUT-01/02/03/04)

Source: 04-CONTEXT.md decisions.

- **Container:** fixed `100vw × 100vh`, `backdrop-filter: blur(12px)`, background `rgba(0,0,0,0.90)`
- **Terminal window:** centered `680px × 480px` panel (desktop), `border: 1px solid rgba(0,255,255,0.3)`, `background: #000000`
- **Terminal chrome bar:** `height: 28px`, background `rgba(20,20,20,0.98)`, three decoration dots (red/yellow/green, 12px circles, `gap: 8px`, left-inset `16px`)
- **Terminal body:** `padding: 16px`, `overflow-y: auto`, font `14px Roboto Mono`, color `#00FFFF` for commands and output, `line-height: 1.6`
- **Prompt marker:** `$` in `--neon-cyan`, `letter-spacing: 0.1em`
- **Cursor blink:** CSS `animation: blink 1s step-end infinite` on a `▌` character, color `#00FFFF`, between commands
- **Typewriter speed:** 35ms per character, 400ms pause between command groups — source: 04-CONTEXT.md
- **Scroll behavior:** terminal body auto-scrolls to bottom as text is typed (`scrollTop = scrollHeight` after each character append)
- **Contact links:** `<a>` tags, color `#00FFFF`, `text-shadow: 0 0 8px #00FFFF`, `text-decoration: underline`, `cursor: pointer`, `target="_blank"`
- **Data source:** `src/data/contact.js` — executor must prompt user to fill github, linkedin, email

### Skills Sector — Network Graph (SKILLS-01/02/03/04/05)

Source: 04-CONTEXT.md decisions.

- **Container:** fixed `100vw × 100vh` div, `backdrop-filter: blur(12px)`, background `rgba(0,0,0,0.90)`
- **Canvas:** HTML Canvas 2D, fills full container via `width: 100%; height: 100%` (CSS) — resolution set to `window.innerWidth × window.innerHeight` on mount and resize
- **Tier 1 layout:** 5 nodes in radial arrangement, center of canvas, radius 200px, evenly spaced at 72° apart
- **Tier 1 node:** filled circle, `radius: 30px`, fill = `skillCategories[i].color` (from `src/data/skills.js`)
- **Tier 1 label:** Roboto Mono 14px, drawn below node center at `y + radius + 14px`, color = `skillCategories[i].color`
- **Hub edges:** faint lines from canvas center to each Tier 1 node, `rgba(0,255,255,0.2)`, `lineWidth: 1`
- **Tier 1 click — expansion (SKILLS-03):**
  - Light-racer animation: a point travels from Tier 1 node outward at ~340px/s along polar-offset direction (one per skill), leaving a fading trail in `rgba(0,255,255,0.6)`, duration 0.6s per racer
  - On racer arrival: Tier 2 node spawns — filled circle `radius: 18px`, fill = parent node color at 70% opacity (`rgba(r,g,b,0.7)`)
  - Tier 2 label: Roboto Mono 12px, drawn beside node (right-offset), color = parent category color
  - Connecting line from Tier 1 to Tier 2: `rgba(category.color at 0.4 opacity)`, `lineWidth: 1`
  - All racers fire simultaneously; nodes appear staggered by their individual travel time
- **Tier 1 second click — collapse (SKILLS-04):** reverse animation — trails retract toward Tier 1 node, Tier 2 nodes disappear on retraction. Duration: 0.4s
- **Hover (SKILLS-05):** cursor within 35px of any node center triggers:
  - Brightness pulse: redraw node fill at full alpha (no transparency)
  - Connected edges: `lineWidth: 2` + full color alpha
  - Label: always shown on hover (even if collapsed)
  - Cursor: `cursor: pointer` on canvas via JS `canvas.style.cursor`
- **Canvas resize:** `ResizeObserver` or `window.resize` listener redraws at new dimensions; node positions recalculate from new center

### Mobile Graceful Degradation (NFR-02)

Source: 04-CONTEXT.md decisions.

- **Breakpoint:** `<768px` — detected via `window.matchMedia('(max-width: 767px)')` with change listener
- **OrbitControls:** disabled — `enabled={!isMobile}` prop on OrbitControls in `Scene.jsx`
- **Grid background:** replaced with static CSS `background-image: repeating-linear-gradient` grid pattern — no Three.js computation. Pattern: `rgba(0,255,255,0.08)` lines on `#000000`, 40px grid pitch
- **Gateway panes:** replaced by 3 vertically stacked 2D mobile cards in a scrollable `div` (full viewport height, `overflow-y: auto`)
  - Card: `border: 1px solid rgba(0,255,255,0.4)`, `background: rgba(0,255,255,0.05)`, `padding: 24px 16px`, `min-height: 120px`
  - Card label: TR2N font, `clamp(1.2rem, 5vw, 2rem)`, color `#00FFFF`, with the same `>_` prefix convention
  - Tap: opens corresponding sector (same 2D overlay components — no changes needed for mobile)
- **Sector overlays:** same components as desktop — already mobile-responsive via fixed full-viewport sizing
- **Boot sequence:** scales to fit via `clamp` font sizes (already implemented from Phase 1)

### Sector Close / HUD Return

- **Mechanism:** top-left HUD disc click (existing NAV-01/NAV-02, Phase 2)
- **Visual feedback:** disc hover state — `box-shadow` brightens on mouse enter (pre-existing behavior)
- **No additional "close X" button** — HUD disc is the sole return affordance (avoids double-affordance confusion, consistent with Phase 1 disc-click removal decision)

## Performance Constraints

| Constraint | Value | Enforcement |
|------------|-------|-------------|
| Target FPS (desktop) | 60 FPS stable — existing NFR-03, already passing | Sector overlays are HTML (off WebGL thread) — no GPU cost |
| Canvas texture update (Skills graph) | Triggered only on: click, hover state change, window resize — no continuous rAF loop in idle state | Prevents unnecessary CPU usage when graph is static |
| Light-racer animation | rAF loop active only during expansion/collapse transitions (max 0.6s each) | Loop cancelled via `cancelAnimationFrame` immediately after transition completes |
| Backdrop-filter | `blur(8px)` on Projects, `blur(12px)` on About/Skills — GPU composited | No JS involvement; browser handles compositing |
| Mobile WebGL | OrbitControls disabled; static CSS grid replaces GridFloor computation | Reduces GPU draw calls to near zero on mobile |
