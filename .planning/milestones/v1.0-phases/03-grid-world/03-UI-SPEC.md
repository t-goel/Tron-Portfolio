## Spacing Scale

Declared values (multiples of 4 only). Phase 3 is primarily 3D WebGL — spacing tokens govern overlay HUD elements and Canvas 2D texture layout, not traditional DOM layouts.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps in HUD, Canvas 2D glyph cell padding |
| sm | 8px | HUD element gaps (disc → name text), canvas text margin |
| md | 16px | HUD block padding (top-left disc container), canvas label inset |
| lg | 24px | Vertical spacing between HUD rows |
| xl | 32px | Canvas pane texture character grid row height |
| 2xl | 48px | Reserved — not used in Phase 3 |
| 3xl | 64px | Reserved — not used in Phase 3 |

Exceptions:
- HUD disc is fixed at 40×40px (pre-existing from Phase 2, src/index.css `.hud-disc`) — not on the 8-point scale but must not be altered.
- Canvas 2D texture resolution for pane surfaces: 240×160px offscreen canvas. Internal grid cells are 8×8px per character — aligns to xs (4px) × 2. Source: 03-CONTEXT.md decisions.
- Three.js world units are independent of CSS px spacing. Pane geometry (5 × 3.5 units), formation radius (8 units), camera position (y=8, z=14) are not mapped to CSS spacing tokens.

## Typography

### DOM / HUD Layer (Design-System Type Scale)

Exactly 4 sizes. The Canvas 2D layer uses its own render-context sizes documented separately below — those are NOT part of the design-system type scale.

| Role | Size | Font | Weight | Line Height | Letter Spacing |
|------|------|------|--------|-------------|----------------|
| Display (name) | clamp(2rem, 5vw, 4rem) | TR2N | 400 (normal) | 1.2 | 0.3em |
| CTA button | clamp(1.2rem, 3vw, 2rem) | TR2N | 400 | 1.0 | 0.35em |
| Label (subtitle) | clamp(0.7rem, 1.5vw, 1rem) | Roboto Mono | 400 | 1.4 | 0.5em |
| HUD micro | 14px | Roboto Mono | 400 | 1.4 | 0.2em |

Hierarchy (largest to smallest): Display > CTA button > Label > HUD micro. Each step is a meaningful size reduction. Two weights used: 400 (normal) only — TR2N and Roboto Mono are used at regular weight throughout this phase.

Source: src/components/UI/TitleOverlay.jsx and EnterButton.jsx inline styles — these values are locked from Phase 1/2 implementation.

### Canvas 2D Pane Texture Layer (Render-Context Sizes — Not Design-System Type Scale)

These sizes are set via `ctx.font` on an offscreen `<canvas>` element. They are scoped to the Canvas 2D rendering context and do not participate in the DOM type hierarchy above. Canvas does not inherit CSS font-face declarations automatically — ensure fonts are pre-loaded via DOM before drawing (FontFace API or a hidden DOM element referencing each font).

| Role | ctx.font String | Usage |
|------|----------------|-------|
| Idle stream characters | `"8px 'Roboto Mono', monospace"` | Random hex/ASCII glyph cells on 8×8px grid — active in idle and decrypt-transition states |
| Decrypting characters | `"8px 'Roboto Mono', monospace"` | Transition — random glyph → label character (same size as idle stream) |
| Revealed pane label | `"16px 'TR2N', sans-serif"` | Final state after decrypt completes: `>_ ABOUT_ME`, `>_ SKILLS`, `>_ PROJECTS`, drawn centered, color #00FFFF |

## Color

Source: src/index.css CSS variables. All values are locked from project design system.

| Role | Hex | CSS Variable | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | #000000 | --void-black | Full WebGL canvas background, pane interior fill, canvas texture background |
| Secondary (30%) | #00FFFF (at low opacity) | --neon-cyan | Pane glass body (opacity 0.15–0.25), grid floor illumination, OrbitControls visual field |
| Accent (10%) | #00FFFF (full brightness) | --neon-cyan | Pane EdgesGeometry border lines, hover-state revealed label text, grid light pulse ring, idle stream characters (primary glyph color) |
| Secondary accent | #FF5E00 | --neon-orange | Idle stream characters (secondary/dim glyph color — roughly 30% of glyphs use this to create visual variation) |
| HUD accent | #FF0000 | --crimson-red | HUD home disc border/glow (pre-existing, Phase 2). Not used in Phase 3 new components. |
| Text | #F0F0F0 | --off-white | Dim idle stream characters (remaining ~30% of glyphs), HUD subtitle text |

Accent (#00FFFF full brightness) is reserved for exactly these elements:
1. Pane EdgesGeometry border lines (emissive cyan frame around each glass pane)
2. Revealed label text on pane surface after decrypt completes
3. Grid light pulse ring expanding from pane base on hover
4. Idle stream characters (primary color, ~40% of glyph draws)
5. OrbitControls — no visual change, but grid floor emissive color is #00FFFF from Phase 2 (GridFloor.jsx)

Pane glass material:
- `MeshStandardMaterial`, `transparent: true`, `opacity: 0.18` (midpoint of 0.15–0.25 range)
- `color: #00FFFF`
- `emissive: #00FFFF`
- `emissiveIntensity: 0.04` (barely glowing — glass, not light source)
- `side: THREE.DoubleSide`

Pane edge overlay:
- `LineSegments` on `EdgesGeometry`
- `LineBasicMaterial`, `color: #00FFFF`, `transparent: true`, `opacity: 0.8`

Grid pulse ring:
- `MeshBasicMaterial`, `color: #00FFFF`, `transparent: true`
- Start opacity: `0.8` → end opacity: `0.0` over 0.8s as ring expands radius 0.5 → 5

## Interaction Contract

### Pane Idle State

- Each pane renders a Canvas 2D `CanvasTexture` (240×160px) updated at ~8 fps (throttled — redraw only if >125ms elapsed since last draw in `useFrame`)
- Canvas fill: `#000000` on each frame (full clear)
- Character grid: 8×8px cells covering the canvas (30 columns × 20 rows = 600 cells)
- Each cell draws one character from the idle character set (see Copywriting Contract)
- Color per character: ~40% Neon Cyan (#00FFFF), ~30% Neon Orange (#FF5E00 at 60% opacity = `rgba(255,94,0,0.6)`), ~30% Off-White (#F0F0F0 at 40% opacity = `rgba(240,240,240,0.4)`)
- Additionally: 3–5 random line segments drawn per frame as faint cyan lines (`rgba(0,255,255,0.15)`) to simulate chaotic wireframe art
- Each of the 3 panes has a unique seed that offsets its random character density and line angle range, ensuring visual differentiation

### Hover State — Decrypt Transition

- Triggered by `onPointerEnter` on the pane mesh
- `decryptProgress` runs 0 → 1 over exactly 1.2 seconds (linear — no ease; the character-by-character reveal creates its own perceived ease)
- Each character cell: if `cellIndex / totalCells < decryptProgress` → draw the correct label character; else draw a random glyph
- Characters reveal left-to-right, top-to-bottom (cell index order)
- Once `decryptProgress > 0.8`: background wireframe lines snap to a symmetric 4×4 crosshatch grid in place of random lines
- Final revealed label: pane-specific string (see Copywriting Contract), drawn at 16px TR2N, centered in the canvas, color #00FFFF
- Simultaneously: fire one grid light pulse animation at pane's XZ base position

### Hover State — Grid Light Pulse

- Spawn: on `onPointerEnter`, activate pane's ring ref (pool of 3)
- Geometry: `THREE.RingGeometry` with inner radius growing from 0.5 to 5, outer radius = inner + 0.1 (thin line ring)
- Position: XZ of pane base, Y = -3 (matches GridFloor Y position, source: 03-CONTEXT.md + GridFloor.jsx)
- Duration: 0.8 seconds
- Animation: `radius 0.5 → 5` (linear), `opacity 0.8 → 0` (linear)
- Second pulse: if user re-hovers before first pulse is done, second pulse fires when first is at 50% progress (don't wait for full completion)

### Hover Leave — Re-scramble

- Triggered by `onPointerLeave`
- Re-scramble: reverse decrypt — `decryptProgress` runs 1 → 0 over 0.6 seconds
- Characters return to random glyph draws as progress recedes
- No additional grid pulse on leave

### Pane Rise Animation (Phase Entry)

- Panes start at y = -5 (below grid floor at y = -3)
- Rise to final position y = 1.5 over 1.5 seconds
- Easing: `power2.out` (GSAP) — fast initial rise, gentle settle at top
- Stagger: panes rise together (simultaneous, not staggered) — the formation should feel like one event, not a sequence
- Trigger: when `phase` transitions to 3 in Zustand state

### Camera / OrbitControls

- Default camera: `position [0, 8, 14]`, `fov: 60`, `lookAt [0, 1, 0]`
- `enablePan: false`
- `maxPolarAngle: Math.PI / 2.1` (cannot orbit below grid horizon)
- `minDistance: 6`, `maxDistance: 22`
- `enableDamping: true`, `dampingFactor: 0.05`
- Active only when `phase >= 3`

### Billboarding

- Each pane: `useFrame` sets `mesh.rotation.y = Math.atan2(camera.position.x - mesh.position.x, camera.position.z - mesh.position.z)`
- No X or Z rotation modification — Y-axis only
- Applied every frame (no throttle — smooth billboard is critical for 3D believability)

## Triangle Formation Layout

| Pane | Position (x, y, z) | Label | Notes |
|------|-------------------|-------|-------|
| Front-center | (0, 1.5, -8) | `>_ PROJECTS` | Faces camera default — most prominent, matches priority-of-access goal |
| Rear-left | (-6.93, 1.5, 4) | `>_ ABOUT_ME` | ~120° clockwise from front |
| Rear-right | (6.93, 1.5, 4) | `>_ SKILLS` | ~240° clockwise from front |

Derivation: radius 8 units, equilateral triangle. Front pane at angle 270° (directly toward camera default z=14). Left: 270°+120°=30° → (8·cos30°, 8·sin30°) = (6.93, 4). Right: 270°+240°=150° → (-6.93, 4). Projects front-center per Claude's discretion note in 03-CONTEXT.md.

Source: 03-CONTEXT.md decisions (formation radius 8, dimensions 5×3.5, discretion: which pane faces front).

## Performance Constraints (NFR-03)

| Constraint | Value | Enforcement |
|------------|-------|-------------|
| Target FPS | 60 FPS stable on modern desktop | Draw calls minimized — 3 panes + 3 edge overlays + 3 ring refs = max 9 meshes |
| Canvas texture update rate | 8 fps max (125ms throttle in useFrame) | Prevents per-frame texture upload CPU cost |
| Ring pulse pool size | 3 refs maximum (one per pane) | No dynamic geometry creation during animation |
| Billboard update | Every frame in useFrame | Required for smooth tracking — rotation.y calc is O(1) per frame |
